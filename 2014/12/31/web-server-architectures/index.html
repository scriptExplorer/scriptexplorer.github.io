    web服务端架构

    Published: December 31, 2014 by scriptExplorer

    • Categories:
    • architecture 1
    • popular 1
    • Tags:

    在过去的几年里, 我所在的团队做了几个比较大的web项目, 累积了一些web架构经验总结于此, 本文偏向于的宏观设计.

    如图:

    • CDN 和 Reverse Proxy 一样,都是对内容进行缓存. 不同之处在于CDN部署在网络提供商机房, 而反向代理部署在网站中心机房. 常用的反向代理软件有: nginx haproxy apache squid 我们可以配合 keepalived 实现高可用. 反向代理 我们使用的是 haproxy

    • Load Balancing 是指多台服务器提供相同服务,为了提高系统 Availability 常用软件有: haproxy F5 nginx LVS 我们团队使用 haproxy 和 LVS

    • Web Server 就是我们的网站了, 一般使用 nginx apache tomcat jetty resin

    • Service Server 基于Micro services理念,将不同的功能\服务分割开, 构造成高内聚低偶尔的独立模块.便于维护提高扩展力.

    • File System 文件系统,一般系统常用的有RAID Hadoop Fs TFS

    • RDBMS 关系型数据库 一般使用 Msql Oracle PostgreSQL

    • NoSQL 源自互联网的技术手段, 一般都有更好的伸缩性, 我们一般使用 MongoDB CouchDB HBase Cassandra Neo4j

    • Cache Server 缓存服务器一般使用 Memcached Redis

    • Search Server 搜索服务器我们一般使用 Lucene/Solr ElasticSearch Sphinx

    • Computing Server 一般都会使用MapReduce策略

    Web架构

    在设计一个web项目的架构时，我一般从以下几个方面去考虑。

    横向分层

    分层架构 说明
    视图层 页面展示
    应用层 业务逻辑
    服务层 模块封装
    数据层 数据访问

    纵向分割

    也就是我们常说的Micro services (cell/set架构), 原理是将一个大型系统拆分为N个小型系统。 每个系统通过接口进行通讯.好处是高内聚低偶尔，提高扩展能力.

    分布式

    这里的分布式主要是指 Sharding, 分布式的好处就是提高了服务器的 scale out能力， 而其关键点就是 stateless。 07年我在做股票平台时,当时 使用了tomcat的集群节点广播复制(jboss的配对复制等session状态复制策略情况也类似) 刚开始效果还不错, 后来发现主要问题是 session复制成本太高, 系统不能通过增加更多的机器来达到良好的水平伸缩，因为集群节点间session的 通信会随着节点的增多而开销增大. 因此想要得到更好的系统伸缩性, 就要保证系统的无状态性. 如果做到系统的无状态性呢? 构建passport 服务, 统一用户状态管理.

    集群

    集群主要指的是 Replication. 提高系统的并发能力和 Available能力。

    缓存

    一般来讲当读写比例在2:1以上，我们才会考虑去缓存。 #### 页面缓存 一般静态资源(JS, css, image, html) 会缓存到 CDN 和 反向代理服务器上， 减小WebServer压力.

    数据缓存

    将数据缓存到 本地缓存/分布式缓存 服务器上，以减小数据库压力。

    异步

    业务处理使用pub/sub模式， 可加快服务器响应速度。 请求处理使用pub/sub模式， 可以控制并发峰值。

    自动化

    测试自动化，发布自动化，自动化监控报警, 自动化失效恢复。

    安全

    XSS攻击， 注入攻击， CSRF 攻击， 用户信息加密， 传输加密, 反垃圾信息。

    Other

    我们可以参考一些指标来衡量数据架构,看看是否满足高并发低延迟的目标。

    操作 时间
    打开一个网站 3s内算正常
    RDBMS使用索引查询一条记录 15 ms
    机械硬盘一次寻址定位 8~10ms
    机械硬盘顺序读取1MB 20~25ms
    SSD顺序读取1MB 0.6ms
    Redis读取一个数据 0.5ms
    LocalCache读取一个数据 15微秒
    Java native invoke method 5 微秒
